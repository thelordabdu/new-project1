## Architecture decisions

### ngrok tunnels port 3000 (frontend), not 8000

The Whoop developer app has `https://polygraph-could-liqueur.ngrok-free.dev` registered as the OAuth redirect base. ngrok tunnels the **frontend** (port 3000), not the backend (8000).

The OAuth callback therefore hits Next.js first. A proxy route was added to forward it to the backend:

**`frontend/app/api/v1/oauth/whoop/callback/route.ts`**
```typescript
export async function GET(request: NextRequest) {
  const params = request.nextUrl.searchParams.toString()
  const backendUrl = `${BACKEND_URL}/api/v1/oauth/whoop/callback?${params}`
  const response = await fetch(backendUrl, { redirect: 'manual' })
  if ([301, 302, 307, 308].includes(response.status)) {
    const location = response.headers.get('location')
    if (location) return NextResponse.redirect(location)
  }
}
```

### API_BASE_URL controls the redirect_uri

The fork builds `redirect_uri` from `settings.api_base_url`. Default is `http://localhost:8000`, which doesn't match what Whoop has registered.

Fix: `API_BASE_URL=https://polygraph-could-liqueur.ngrok-free.dev` in `backend/config/.env`.

## Bugs fixed

### Bug 1 — FernetDecryptorField validation error

**Symptom:** Backend crashed on startup with Fernet key validation error.
**Cause:** `MASTER_KEY` was 43 chars (missing trailing `=`). Fernet keys must be exactly 44 chars, base64-encoded, ending in `=`.
**Fix:** Regenerate:
```python
from cryptography.fernet import Fernet
print(Fernet.generate_key().decode())
```

### Bug 2 — `after_commit` hook accessed ORM attributes after session closed

**Symptom:** Worker crashed with `InvalidRequestError` when saving workout data.
**Cause:** SQLAlchemy's `after_commit` fires when the session is already committed. Accessing ORM attributes (lazy-loading) at this point raises `InvalidRequestError`.
**Fix:** Extract all needed values into plain Python scalars **before** registering the listener. Fixed in `event_record_service.py` via a `_emit_event_record_webhook_scalars()` static method that takes only plain values.

### Bug 3 — Prepared statement errors with Supabase PgBouncer

**Symptom:** `psycopg.errors.InvalidSqlStatementName: prepared statement "_pg3_N" does not exist`
**Cause:** Supabase uses PgBouncer in **transaction mode**. PgBouncer resets connection state between transactions, invalidating per-connection prepared statement caches.

**Critical psycopg3 gotcha:**
- `prepare_threshold=0` → "prepare on first use" (DOES NOT disable prepared statements)
- `prepare_threshold=None` → actually disables server-side prepared statements

**Fix in `backend/app/database.py`:**
```python
engine = create_engine(
    settings.db_uri,
    pool_pre_ping=True,
    pool_size=20,
    max_overflow=30,
    pool_timeout=30,
    pool_recycle=3600,
    connect_args={"prepare_threshold": None},  # None = disabled; 0 = prepare on first use (wrong)
)
async_engine = create_async_engine(
    settings.db_uri,
    connect_args={"prepare_threshold": None},
)
```

**Historical note:** When Celery worker was active, changing `database.py` required rebuilding the worker image — `docker compose restart` was not enough. Current v0.2.1 runtime uses only the `backend` service; see `docs/docker-cheatsheet.md`.

### Bug 4 — OAuth callback 400: foreign key violation

**Cause:** The fork's `user` table had no row for the Supabase auth user.
**Fix:** Insert once in Supabase SQL editor:
```sql
INSERT INTO "user" (id) VALUES ('64278722-3f20-479e-8010-0b9afbb16ba0');
```

## How to trigger a historical sync

### 1. Get a developer JWT (expires in 1 hour)
```bash
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=kutbiamf@gmail.com&password=admin123"
```
Use the `access_token` from the response.

### 2. Create an API key (one-time — key persists in DB)
```bash
curl -X POST http://localhost:8000/api/v1/developer/api-keys \
  -H "Authorization: Bearer <JWT>" \
  -H "Content-Type: application/json" \
  -d '{"name": "dev"}'
# Returns: {"id": "sk-..."}
```
Current dev API key: `sk-e027b7f8172594fb13bd2254347e2a73`

### 3. Trigger historical sync
```bash
curl -X POST "http://localhost:8000/api/v1/providers/whoop/users/64278722-3f20-479e-8010-0b9afbb16ba0/sync/historical?days=30" \
  -H "X-Open-Wearables-API-Key: sk-e027b7f8172594fb13bd2254347e2a73"
```

### 4. Verify data in Supabase
```sql
SELECT
  (SELECT COUNT(*) FROM health_score WHERE user_id = '64278722-3f20-479e-8010-0b9afbb16ba0') AS health_scores,
  (SELECT COUNT(*) FROM event_record er
   JOIN data_source ds ON ds.id = er.data_source_id
   WHERE ds.user_id = '64278722-3f20-479e-8010-0b9afbb16ba0') AS event_records;
```
