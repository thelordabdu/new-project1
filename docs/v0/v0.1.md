# v0.1 — Signal Processing

**Goal:** Fix RMSSD computation. Filter artifacts. Get our HRV number close to what Whoop shows.  
**Success criteria:** Our computed RMSSD is within ~10% of Whoop app HRV display (manual comparison).  
**Scope:** Frontend only. No backend, no DB, no API.

---

## What's wrong with v0.0

1. **RMSSD computed across packet boundaries** — the last R-R of one packet and the first R-R of the next are diffed as if they're consecutive beats. They may not be. This inflates RMSSD to 200–400 ms instead of a realistic 20–80 ms.
2. **No artifact filtering** — motion spikes (e.g. 1271 ms in the middle of 580–700 ms values) are included in the computation.
3. **No gap detection** — if BLE drops for 3 seconds and reconnects, intervals across that gap are still diffed.

---

## What v0.1 builds

### 1. Fix RMSSD computation

RMSSD must only be computed within a single packet's R-R array, never across packets.

**Current (wrong):**
```typescript
const allRR = samples.flatMap(s => s.rr_intervals);
const diffs = allRR.slice(1).map((v, i) => Math.pow(v - allRR[i], 2));
```

**Fixed:** compute squared diffs only within each packet, then pool all within-packet diffs together before taking sqrt mean.

```typescript
function computeRMSSD(samples: RRSample[]): string {
  const squaredDiffs: number[] = [];
  for (const sample of samples) {
    const rr = sample.rr_intervals;
    for (let i = 1; i < rr.length; i++) {
      squaredDiffs.push(Math.pow(rr[i] - rr[i - 1], 2));
    }
  }
  if (squaredDiffs.length < 2) return 'not enough data';
  const rmssd = Math.sqrt(squaredDiffs.reduce((a, b) => a + b, 0) / squaredDiffs.length);
  return rmssd.toFixed(1) + ' ms';
}
```

### 2. Artifact filtering

Use `cleanRR()` from `lib/signal/filter.ts` on each packet's R-R array before any computation.

`cleanRR()` applies two filters in sequence:
- `filterPhysiological()` — removes values outside 273–2000 ms
- `filterArtifacts()` — removes values >20% from local median

Apply per packet, not across the full buffer.

### 3. Packet gap detection

Track `ts` (timestamp) between consecutive samples. If the gap between two packets is >2000 ms, treat them as non-contiguous — never diff R-R values across that boundary.

Store a `gap` flag on each sample in the buffer. Skip inter-packet diffs where either packet is flagged.

### 4. Signal quality indicator

After filtering, compute:
```
quality = filtered_count / raw_count * 100
```

Display as a percentage. If quality drops below 80%, show a warning ("poor signal — sit still").

### 5. Per-packet RMSSD with rolling average

Instead of one global RMSSD over the whole buffer, compute RMSSD per packet (within-packet diffs only), then display a rolling average of the last 10 per-packet RMSSD values.

This gives a smoother, more stable number — closer to what Whoop computes.

### 6. Session export button

Add a Download JSON button that triggers:
```typescript
const blob = new Blob([JSON.stringify(samples, null, 2)], { type: 'application/json' });
const url = URL.createObjectURL(blob);
// trigger download
```

No more manually copying from the `<pre>` block.

### 7. Auto-reconnect on drop

If `device.gatt` disconnects unexpectedly, attempt one automatic reconnect before requiring manual action. Show status: `reconnecting...` during the attempt.
