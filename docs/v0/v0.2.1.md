# v0.2.1 — Tech Stack Config Cleanup

**Goal:** Make the active development stack match what the project needs right now.

**Status:** Complete

---

## Decision

The active v0 stack is:

```text
Frontend -> FastAPI backend -> Supabase Postgres
```

Docker stays as an optional backend runtime wrapper. Supabase Postgres stays external. Celery and Redis are deferred until the app needs real background jobs.

---

## Why

The previous compose setup ran FastAPI, a Celery worker, and Celery beat. That is production-shaped, but it creates extra moving parts before the product pipeline is stable.

For v0.2.1, the backend only needs to:
- serve API routes
- handle Whoop OAuth callbacks
- run manual sync/backfill endpoints
- talk to Supabase Postgres

Background jobs become useful later when v0.4 introduces nightly comparator work or when provider syncs need retries and queueing.

---

## What Changed

| Area | Change |
|---|---|
| Docker Compose | Active service is now `backend` only |
| Supabase Postgres | Remains external through `backend/config/.env` |
| Redis | Removed from active runtime |
| Celery worker/beat | Removed from active runtime |
| Docs | Added `docs/docker-cheatsheet.md` |

---

## Deferred, Not Forgotten

Keep background-job structure in mind for later backend architecture:

```text
app/jobs/
├── celery_app.py
├── sync_jobs.py
└── comparator_jobs.py
```

Likely reintroduction point:
- v0.4 comparator nightly task
- webhook-triggered snapshot refreshes
- provider sync retry handling

---

## Commands

Start backend:

```bash
docker compose up -d backend
```

Logs:

```bash
docker compose logs -f backend
```

Rebuild after dependency or Dockerfile changes:

```bash
docker compose build --no-cache backend
docker compose up -d backend
```

Run without Docker:

```bash
cd backend
uv sync
uv run uvicorn app.main:api --host 0.0.0.0 --port 8000 --reload
```

Full Docker notes: `docs/docker-cheatsheet.md`
